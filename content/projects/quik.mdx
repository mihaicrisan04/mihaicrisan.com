Inline AI in your terminal. Hit `Cmd+Enter`, ask anything, get an answer streamed in place. Built because I wanted Warp's inline-AI feature without giving up my actual terminal.

The friction it removes: every "quick question" — a flag I half-remember, a regex escape, a git error message I want explained — used to mean alt-tabbing to a browser, pasting, waiting, alt-tabbing back. Quik kills the round trip and keeps the answer in the same buffer as the command it's about.

## Install

```sh
brew install mihaicrisan04/tap/quik
```

Then add to `~/.zshrc`:

```sh
source "$(brew --prefix)/share/quik/quik.plugin.zsh"
```

Needs [`claude`](https://docs.claude.com/en/docs/claude-code) or [`codex`](https://github.com/openai/codex) on `PATH`.

## How the keybinding works

`quik` listens for the control byte `0x1f`. You map `Cmd+Enter` → that byte in your terminal of choice (Ghostty, iTerm, Kitty, WezTerm — see the wiki). The plugin handles everything else.

## Design notes

The Rust core stays out of the hot path: the zsh plugin runs entirely in your shell session, only shelling out to the inference CLI when you actually ask. That means zero overhead on shell startup and no background daemons.
