a tiny macOS CLI that AirDrops files and URLs from your terminal. multiple files in one transfer. developed and tested on macOS 26.

```sh
ad file.pdf
ad a.md b.png c.mov
ad https://example.com
```

## why

AirDrop is the fastest way to move a file off a Mac, but the share-sheet dance — focus Finder, navigate, right-click, share, pick the device — pulls you out of whatever you were doing. i'm in the terminal all day. i wanted `ad file.pdf` and done.

## install

```sh
brew install mihaicrisan04/tap/airdrop
```

both methods put `ad` on your `PATH`.

## why this one doesn't hang

every other CLI airdrop tool i tested hangs after the picker dismisses — the system kills the share session before the transfer confirms. this one ships as a proper `.app` bundle, so `sharingd` keeps the share session alive until the receiving side accepts.

## what's inside

the CLI is a thin shim around an `NSSharingService` with `NSSharingServiceNameSendViaAirDrop`. the hard part wasn't the API — it was figuring out the lifecycle: you need a running app process (not a one-shot CLI binary) for `sharingd` to consider the share valid.
