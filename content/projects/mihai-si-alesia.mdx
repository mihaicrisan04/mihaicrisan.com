A tiny photo gallery I built for me and alesia. Same set of photos, two different ways of looking at them.

## Scattered prints

The home page drops polaroid-style prints across a light canvas. The placement isn't random — it uses an R2 quasirandom sequence so the distribution stays visually even no matter how many photos you throw at it, with a splitmix32 PRNG seeding the per-photo rotation and offset so the result is stable across reloads.

There's a hidden control panel: type `mia` on the keyboard and you get sliders for canvas size, tightness, scatter, rotation, and photo size. Mostly there because I kept wanting to tweak the numbers without redeploying.

## Analog memory

The `/alt` route is a polaroid grid with a film grain overlay. Each polaroid has a click-to-zoom modal that uses a FLIP animation (First-Last-Invert-Play) for a smooth transition from grid position to full-screen, then back when dismissed. Portal-rendered, escape-key and click-outside dismissal, scroll lock — the boring details that make modals feel right.

## Notes

The whole thing is static-exported (`output: "export"`) so it deploys as plain HTML/CSS/JS — no server, no API, instant loads. Photos live in `/public` as pre-optimized WebP files generated by a small Sharp script.
